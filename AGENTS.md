# AGENTS.md — shadcnui-foundry

This file provides guidance for AI coding agents (GitHub Copilot, Claude, GPT, etc.)
working in this repository.

## Project Purpose

shadcnui-foundry is a TypeScript monorepo that ingests shadcn/ui component definitions
and generates idiomatic implementations for React, Vue, Svelte, Angular, and Lit.

## Monorepo Structure

```
/apps
  /docs                    # Documentation site (future)
  /playground-react        # React component playground (future)
  /playground-vue          # Vue component playground (future)
/packages
  /ir                      # Intermediate representation schemas (Zod)
  /core                    # Pipeline interfaces + orchestration
  /registry-ingest         # shadcn/ui registry fetching + caching
  /analyzer                # Raw artifact → ComponentIR normalization
  /react                   # React transformer + emitter
  /vue                     # Vue transformer + emitter
  /svelte                  # Svelte transformer + emitter
  /angular                 # Angular transformer + emitter
  /lit                     # Lit/Web Components transformer + emitter
  /tokens                  # Design token extraction + serialization
  /a11y-rules              # Accessibility validation rules
  /test-utils              # Shared test fixtures + assertions
  /cli                     # CLI entry point (commander)
/tooling
  /typescript-config       # Shared tsconfig bases
  /biome-config            # Shared Biome config
  /vitest-config           # Shared Vitest config
```

## Pipeline Architecture

```
[Registry] → Ingest → Analyze → Transform → Emit → [Output Files]
                                               ↓
                                           Validate (a11y-rules)
```

Every stage returns `PipelineResult<T>` — never throw from pipeline code.

## Key Rules for Agents

### TypeScript

1. **Always use strict TypeScript** — `exactOptionalPropertyTypes`, `noUncheckedIndexedAccess`
2. **Import types with `import type`** when not using the value at runtime
3. **Always include `.js` extensions** in relative imports (ESM requirement)
4. **Never use `any`** — use `unknown` and narrow explicitly
5. **Never use `!`** non-null assertions in production code
6. **Use Zod** to validate all external data

### Pipeline Code

7. **All pipeline stages return `PipelineResult<T>`** — `{ success: true, data }` or `{ success: false, errors }`
8. **Never silently overwrite files** — check existence and prompt for confirmation
9. **Always set `irHash`** on `GeneratedFile` to `sha256(JSON.stringify(ir))`
10. **Always include provenance metadata** on ingested artifacts

### Generated Files

11. All generated files must begin with:
    ```
    // @generated by shadcnui-foundry — DO NOT EDIT MANUALLY
    // Run `foundry generate <component>` to regenerate
    ```
12. Use `gen:` commit type for commits that update generated files

### Testing

13. Test files: `src/**/*.test.ts` (not `.spec.ts`)
14. Use `createTestComponentIR()` from `@shadcnui-foundry/test-utils` for fixtures
15. Use `expectSuccess()` / `expectFailure()` helpers for `PipelineResult`
16. Never delete or skip existing tests

### Package Boundaries

17. `@shadcnui-foundry/ir` must have **no internal deps** (only `zod`)
18. `@shadcnui-foundry/core` depends only on `ir`
19. Framework packages must only depend on `core` + `ir`
20. `cli` is the only package that orchestrates the full pipeline

## Reusable Prompts

### Ingest a shadcn component

```
Ingest the shadcn/ui component "<NAME>" using the ShadcnRegistryIngester.
Validate the response against ShadcnRegistryItemSchema.
Cache the artifact in .foundry/cache/.
Return a RawRegistryArtifact with provenance metadata.
```

### Analyze to IR

```
Analyze the RawRegistryArtifact for "<NAME>" using ShadcnAnalyzer.
Classify the component category using classifyComponent().
Infer ARIA roles from the component name.
Return a validated ComponentIR.
```

### Generate React component

```
Transform the ComponentIR for "<NAME>" using ReactTransformer.
Emit TSX source using ReactEmitter.
Include the @generated header comment.
Set irHash to sha256(JSON.stringify(ir)).
Output to packages/react/src/components/<name>/<ComponentName>.tsx.
```

### Write a test

```
Write a Vitest test in src/<file>.test.ts.
Use createTestComponentIR() from @shadcnui-foundry/test-utils for fixtures.
Use expectSuccess() / expectFailure() for PipelineResult assertions.
Test both success and failure paths.
```

### Validate accessibility

```
Validate the ComponentIR for "<NAME>" using A11yValidator.
Check all A11Y_RULES.
Return a ValidationReport with issues and passed flag.
Report any A11Y001, A11Y002, A11Y003 violations.
```

## MCP Integration

When MCP (Model Context Protocol) is configured:
- MCP config lives in `.foundry/mcp.json`
- Use shadcn MCP tools to fetch live registry data
- Cache all MCP responses in `.foundry/cache/`
- Fall back to `LocalFileIngester` if MCP is unavailable
- Always validate MCP responses against `ShadcnRegistryItemSchema`

## Agent Safety Checkpoints

Before making any changes, agents MUST:
1. **Read the relevant interface contracts** in `packages/core/src/interfaces/`
2. **Check the IR schema** in `packages/ir/src/schemas/`
3. **Run existing tests** before and after changes
4. **Never modify test snapshots without explicit confirmation**
